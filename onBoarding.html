SOP 0→5 (Mudah Dipahami)

0) Intake & Triage
  • Kumpulkan masukan & bug (form/Slack).
  • Tandai prioritas (Idea/Improvement/Bug Sev1‑3).
  • Tentukan siapa PIC & kapan tindak lanjut.
1) Discovery
  • Wawancara singkat + survei cepat.
  • Cek data (funnel, waktu belajar, drop‑off).
  • Tulis masalah inti & kenapa penting.
2) Definition
  • Buat PRD mini: tujuan, KPI, scope in/out, acceptance criteria, risiko.
  • Susun event plan untuk tracking.
  • Review CPO & CTO (singkat tapi jelas).
3) Delivery
  • Sprint 1–2 minggu. DoR/DoD jelas.
  • QA fungsional + regresi cepat.
  • Tulis changelog & siapkan flag.
4) Launch
  • Rilis bertahap (canary 10%).
  • Monitor error, performa, feedback.
  • Siapkan rollback.
5) Learn
  • Bandingkan hasil vs baseline.
  • Tentukan: lanjut, iterasi, atau hentikan.
  • Dokumentasi singkat supaya mudah di‑hand‑over.

Handshake (Titik cek wajib)

  1. PRD siap & disetujui CPO/CTO.
  2. Desain & API brief jelas + event plan.
  3. Dev kickoff: estimasi, risiko, feature flag.
  4. Pre‑release QA: AC lulus, performa oke, tracking valid.
  5. Canary 10% + monitoring aktif, rollback siap.
  6. Post‑launch readout: hasil vs baseline.

Cara Kerja Sehari‑hari

Ritual

  • Planning (Senin)
  • Product Review (Rabu)
  • Demo + Retro (Jumat)
  • Daily update 3‑liner

Tools

  • Notion/Jira (tracking & dokumen)
  • Figma (desain)
  • GitHub (kode)
  • GA4/Amplitude (analitik)
  • Sentry/Crashlytics (error)

Quality Bar

  • DoR: PRD jelas, desain siap, event plan ada.
  • DoD: QA lulus, tracking valid, changelog, dokumen.
  • Severity: Sev1 (kritis) → war‑room.

Glossary (Istilah penting)

  • WAL — Weekly Active Learners, jumlah user unik yang belajar minggu itu.
  • D7 Retention — persentase user baru yang kembali di hari ke‑7.
  • PRD — Product Requirements Document: tujuan, KPI, scope, AC, event plan.
  • DoR/DoD — Definition of Ready/Done: ceklis sebelum mulai & selesai.
  • Sev1/2/3 — tingkat masalah: kritis/serius/minor.
  • RCA — Root Cause Analysis, analisis akar masalah setelah insiden.

FAQ Ringkas

  • Akses akun (Slack/Drive/Notion/GitHub) diberikan setelah FM.
  • Jam kerja fleksibel (WFA). Utamakan hadir di ritual mingguan.
  • Eskalasi bug: Sev1 (war‑room), Sev2 (≤24 jam), Sev3 (masuk backlog).
  • Update harian: tulis 3‑liner (yesterday/today/blocker).
